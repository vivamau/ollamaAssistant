Ollama Assistant

A companion for Ollama that helps you get the most out of your local LLMs.

Checkout the code on Github and have fun!

Why Ollama Assistant

Ollama, along with LM Studio, was one of the first tools that made it easy to use AI on our computers. Even though Ollama has recently added a graphical interface to make it easier to use, some parts still feel a bit lacking. That’s why I’ve created an alternative interface with features that I think would be handy for everyday users.

Ollama Assistant is designed to be a lightweight and user-friendly application that enhances your experience with local LLMs. It provides a range of features, including prompt management, chat history, and customizable settings, all aimed at making your interactions with language models more efficient and enjoyable.

What is Ollama Assistant?


Ollama Assistant lets you save and organize prompts and link them together with your LLMs, makes it a breeze to manage the LLMs installed, and lets you create LLM‑boosted setups with documents (PDF, txt, Excel, HTML, markdown, etc.) or even with websites' content.

It is a web app built with NodeJS and ReactJS and SQLlite and it depends on a Ollama installation. You can turn it into a desktop app with Electron, just follow the instructions on the code repository.

Ollama Assistant is open source, published under the AGPL 3 license.

Participation

Strong visual design has always been our focus. You are modifying this page, so strong CSS skills are necessary too, but the example files are commented well enough that even CSS novices can use them as starting points. Please see the CSS Resource Guide for advanced tutorials and tips on working with CSS.

You may modify the style sheet in any way you wish, but not the HTML. This may seem daunting at first if you’ve never worked this way before, but follow the listed links to learn more, and use the sample files as a guide.

Download the sample HTML and CSS to work on a copy locally. Once you have completed your masterpiece (and please, don’t submit half-finished work) upload your CSS file to a web server under your control. Send us a link to an archive of that file and all associated assets, and if we choose to use it we will download it and place it on our server.

Contribute

Contributing to Ollama Assistant is a great way to improve your skills and help others in the community. Whether you’re a seasoned developer or just starting out, there are many ways to get involved. You can contribute by reporting bugs, suggesting new features, or even submitting code changes through pull requests on GitHub.

Requirements

Where possible, we would like to see mostly CSS 1 & 2 usage. CSS 3 & 4 should be limited to widely-supported elements only, or strong fallbacks should be provided. The CSS Zen Garden is about functional, practical CSS and not the latest bleeding-edge tricks viewable by 2% of the browsing public. The only real requirement we have is that your CSS validates.

Luckily, designing this way shows how well various browsers have implemented CSS by now. When sticking to the guidelines you should see fairly consistent results across most modern browsers. Due to the sheer number of user agents on the web these days — especially when you factor in mobile — pixel-perfect layouts may not be possible across every platform. That’s okay, but do test in as many as you can. Your design should work in at least IE9+ and the latest Chrome, Firefox, iOS and Android browsers (run by over 90% of the population).

We ask that you submit original artwork. Please respect copyright laws. Please keep objectionable material to a minimum, and try to incorporate unique and interesting visual themes to your work. We’re well past the point of needing another garden-related design.

This is a learning exercise as well as a demonstration. You retain full copyright on your graphics (with limited exceptions, see submission guidelines), but we ask you release your CSS under a Creative Commons license identical to the one on this site so that others may learn from your work.
